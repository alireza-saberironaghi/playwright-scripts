Overview

The Auto App Runner is an internal desktop application for Thirdstream that enables everyone - Business Analysts, Product Managers, QAs, and Developers - to automatically create applications across multiple tenants and environments. No technical knowledge is required, as everything is configured through a user-friendly interface.

🎯 Main Objectives

  • Reduce time and effort in application creating
  • Simplify testing workflows for all team members
  • Increase efficiency across different environments
  • Enable parallel testing to maximize productivity

⭐ Key Features

🏢 Multi-Tenant Support

Create applications across 12 different tenants (Pathwise, Kawartha, Sunshine, Hubert, etc.)

🌐 Environment Selection

Switch between Dev, QA, Test, Staging, and Production

⚡ Parallel Execution

Run multiple tests simultaneously to save time

🔧 Dynamic Configuration

Tenant-specific options automatically adjust based on selection

Getting Started

💻 System Requirements

  • Operating System: Windows 10 or later
  • Network: Thirdstream VPN (required for QA and Dev environments)
  • Memory: At least 4GB RAM (8GB recommended for parallel execution)
  • Internet: Stable connection for downloading scripts and accessing environments
Note: Mac version will be published soon!

⚡ Installation (Less than 5 minutes)

No installation required! The Auto App Runner is a portable application.
  1. Install Node.js:
    • Go to https://nodejs.org/en
    • Click on "Download Node.js (LTS)"
    • Install Node.js (follow the setup wizard with default settings)
  2. Download Auto App Runner:
    • Download the AutoAppRunner zip file
    • Extract the zip file to your desired location
  3. Launch the Application:
    • Double-click Auto App Runner.exe
    • Start using immediately!

Execution Control

🎯 Single Execution

▶️ Run Button

  • Function: Starts a single test with current configuration
  • Requirements: Valid email and cell phone must be entered

⏸️ Pause Button

  • Function: Pauses the currently running test
  • Test waits at current screen
  • Button changes to "▶️ Resume"
  • Browser remains open and interactive

⏹️ Stop Button

  • Function: Immediately terminates the running test
  • Force kills the test process
  • Resets all buttons to initial state
  • Browser closes automatically

⚡ Parallel Execution

Run multiple test configurations simultaneously for maximum efficiency.

Setting Up Parallel Execution

  1. Configure a test in the Configuration Panel
  2. Click "Add to Parallel Run"
  3. Repeat for additional test configurations
  4. Click "Parallel Run" to manage and execute batch

Batch Queue Management

Each batch item displays:

  • Tenant and Environment (e.g., "Sunshine QA")
  • Main User (e.g., "Chrit")
  • Additional Details: Joint applicant, account options, power of attorney details, third party selections

Queue Actions:

  • 🗑️ Remove individual items
  • × Clear All - Remove all items from queue
  • ▶ Start Parallel Execution - Run all tests simultaneously

Execution Process

  1. Verify queue contains desired tests
  2. Click "Start Parallel Execution"
  3. Monitor real-time progress
  4. Use controls: Pause, Resume, Stop

Troubleshooting

🚨 Common Issues and Solutions

"No running tests to pause/stop"

Cause: No test is currently executing

Solution: Start a test first using "Run Test" button

"Playwright Error" or execution failures

Cause: Various reasons (network issues, environment problems, etc.)

Solutions:

  1. Check internet connection
  2. Verify environment is accessible
  3. Try different execution mode (Normal vs Headless)
  4. Contact Alex Saberi if issues persist

❌ Error Messages

Error Message Solution
"Failed to download scripts" • Check internet connection
• Wait and try again (temporary network issue)
"Playwright test failed" • Check output panel for detailed error
• Verify configuration parameters
• Try different execution mode
• Contact development team if persistent

Frequently Asked Questions

💬 General Usage

Do I need to install anything to use Auto App Runner?
You only need to install Node.js. Auto App Runner is a portable application - just download and double-click to run.
Can I run tests on multiple environments simultaneously?
Yes, using parallel execution you can run tests on different environments at the same time.
How often are the test scripts updated?
Scripts update automatically every time you start the application, and you can manually refresh them anytime.
Can I use this on Mac or Linux?
Currently, Auto App Runner is designed for Windows. Mac and Linux support may be added in future versions.

⚙️ Configuration

Why don't I see all the configuration options?
The app only shows options relevant to your selected tenant. Different tenants have different available features.
What's the difference between execution modes?
Normal: Shows browser window (best for watching/debugging)
Headless: Faster, runs in background (best for speed)
Debug: Step-by-step execution (best for troubleshooting)

⚡ Parallel Execution

How many tests can I run simultaneously?
This depends on your system specifications. Start with 2-3 tests and increase based on performance.
Can I add more tests to a batch while it's running?
No, you cannot modify a batch once execution has started. You'll need to wait for completion or stop the batch.
What happens if one test fails in a parallel batch?
Other tests continue running. Failed tests are marked in the results, but don't affect other tests in the batch.

🔧 Technical Issues

What should I do if a test gets stuck?
Use the Stop button to terminate the test, then try running it again. If the issue persists, contact Alex Saberi.
The application is running slowly. What can I do?
Try using Headless mode, close other applications, and ensure your system meets the recommended specifications (8GB+ RAM).

🆘 Support
